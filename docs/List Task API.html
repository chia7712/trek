List Task API

依照使用者指定的條件列出系統處理過的任務

HTTP request
GET /task/list?{query}

query
說明
id
(row key)
  1. 任務id
  2. 支援wildcard pattern：? and *
redirectFrom
  1. 此任務是從何處（dt server）轉傳過來
  2. 支援wildcard pattern：? and *
serverName
  1. 處理此request的伺服器位址
  2. 支援wildcard pattern：? and *
clientName
  1. 發出此request的客戶端位址
  2. 支援wildcard pattern：? and *
from
  1. 資料來源
  2. 支援wildcard pattern：? and *
to
  1. 資料目的地
  2. 支援wildcard pattern：? and *
state
  1. 任務狀態，共四種。1)pending、2)running、3)succeed、4)failed
progress
  1. 進度百分比 (0.0 ~ 1.0)
  2. 可用([min, max])來設定範圍
starttime
  1. 任務起始時間（從任務被接受開始算起）
  2. 可用([min, max])來設定範圍
  3. yyyy-MM-dd'T'HH:mm:ss.SSS
elapsed
  1. 任務的執行時間 (ms)
  2. 可用([min, max])來設定範圍
  3. 負值則忽略
expectedSize
  1. 預期要傳輸的資料大小（bytes）
  2. 可用([min, max])來設定範圍
  3. 負值則忽略
transferredSize
  1. 已經傳輸的資料大小（bytes）
  2. 可用([min, max])來設定範圍
  3. 負值則忽略
limit
  1. 限制回傳的總筆數
  2. <0則代表無限大，預設值100
offset
  1. 資料位移
  2. 預設值0
orderby
  1. 規定回傳資料的排序方式，共有id, redirectFrom, serverName, clientName, state, progress, starttime, elapsed, expectedSize, transferredSize共10種排序
  2. 預設的排序方式根據為id
asc
  1. 預設值true，代表資料遞增排序
  2. 字串(string)用字典排序
  3. 數值和日期用值的大小排序
